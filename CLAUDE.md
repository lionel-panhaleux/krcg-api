# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

FastAPI REST API for VTES (Vampire: The Eternal Struggle) card game data: VEKN card texts + the Tournament Winning Deck Archive (TWDA). A thin web layer over the `krcg` package. No database — all card and deck data is loaded in-memory at startup.

## Commands

`uv` + `just`: `just install`, `just quality`, `just test`, `just serve`
- `just test` runs `quality` first (ruff format check, ruff lint, `ty check`), then the full pytest suite. Run before pushing.
- Single test: `uv run pytest tests/test_card.py::test -vvs`
- `just bump` / `just release` / `just publish` are maintainer release tooling (push to master, tag, publish to PyPI) — don't run them for feature work.

Tests require live internet: `conftest.py` fails the whole session if Google or the KRCG static server is unreachable, because `krcg` fetches card/TWDA data from the network at startup. CI (`.github/workflows/validation.yml`) runs the same quality+test gate on Python 3.11 and 3.14.

## Code Quality

ruff (format + lint) and `ty` (type checker, **all rules set to error** — see `[tool.ty.rules]`). Both must pass. The OpenAPI spec is auto-generated by FastAPI from route decorators and Pydantic models — there's no spec file to hand-edit.

## Architecture

- **`krcg_api/__init__.py`** — `create_app()` builds the FastAPI app; the lifespan handler calls `vtes.VTES.load()` and `twda.TWDA.load()` (network fetch) before serving. ASGI entrypoint: `krcg_api:application`. CORS is wide open (`allow_origins=["*"]`). `main()` runs uvicorn for local/dev.
- **`krcg_api/api.py`** — every route lives here on a single `APIRouter`. Handlers are deliberately thin: they unpack a request model, call into `krcg` (`vtes.VTES`, `twda.TWDA`, `deck.Deck`, `analyzer.Analyzer`), and return `.to_json()`. Domain logic belongs in `krcg`, not here. Root `/` redirects to `/docs` (Swagger UI).
- **`krcg_api/models.py`** — Pydantic request models. Their `Field` descriptions and `json_schema_extra` examples ARE the API documentation shown in Swagger, so keep them accurate when changing behavior.
- **`krcg_api/config.py`** — re-exports `SUPPORTED_LANGUAGES` / `KRCG_STATIC_SERVER` from `krcg.config`.

### Patterns to follow when editing `api.py`

- **Dual response shape (`mode`)**: `/card_search` and `/candidates` return a list of name strings by default, or full card objects when `mode == "full"`. Because the shape varies, these set `response_model=None` and document both shapes manually via `responses=`/`openapi_extra=` in the decorator. Match this when adding mode-aware endpoints.
- **URL-import endpoints** (`/amaranth`, `/vdb`, `/vtesdecks`) are near-identical: read a `url` from JSON or form body and call `deck.Deck.from_url(...)`. They share the `_url_request_body()` helper for OpenAPI examples.
- **Card lookup** accepts either an integer ID or a (URL-encoded, possibly translated) name; missing cards → 404, invalid card names in filters → 400.
- **Language negotiation**: `_parse_accept_language` + `_negotiate_locale` (via `babel`) drive the `Accept-Language` / `lang` handling for `/complete` and `/card_search`.

New endpoints: add the handler + decorator in `api.py`, add request model(s) in `models.py` if needed, add a test in `tests/` (use the session-scoped `client` TestClient fixture from `conftest.py`), then `just test`.

> Note: `README.md` references a `/submit-ruling` endpoint and `GITHUB_USERNAME`/`GITHUB_TOKEN` env vars that are not present in the current `api.py` — treat the code as the source of truth.
